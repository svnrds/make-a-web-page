Lesson 1: The Basics of the Web and HTML

The Web

The Web was invented in the early 1990s. It is a collection of HTML documents with are all stored on servers. These documents are accessed by a user through a browser, which uses HTTP (HyperText Transfer Protocol) to access the servers over the Internet. At the time this lesson was created there were approximately 30 billion HTML pages.

HTML (HyperText Markup Language)

HTML is the lanuage used to display text content and modify how it looks. The "modify how it looks" is called Markup.

Elements are sections of content denoted by tags like: <b>boldnes</b> The entire line is an element, while the < > parts are the tags. Elements are either inline or block. the example of the bold tags above is an example of inline tags. All it does to bold the text between the tags. Block tags on the other hand, set up an "invisible box" around the element. Not all tags need the closing tag, but not including one that is needed can cause the enitre document to be problematic. Lesson 2: Creating a Structured Document with HTML

There are 3 main parts to a web page:

DOM- Document Object Model - a cross-platform and language independent convention for representing and interacting with objects in HTML (and other markup languages). The nodes of every document are organized in a tree structure, called the DOM tree.

  1. HTML

    The standard markup language used to create web pages.

    HTML pages are organized in blocks. You can nest blocks inside each other. When viewing the source code, you can visually see nesting with the wave of indentions in the left margin of the lines. Representing this as a tree structure is known as the DOM.

  2. CSS

    style sheet language used for describing the look and formatting of a document written in a markup language.

Lesson 3: Adding CSS Style to HTML Structure

Repitition is the Devil

As a coder, repition is awful. Writing the same code over and over can lead to bugs with can be hard to track down since it will be hard to find the correct lines of code to squash that were copied and pasted.

Repitition also makes it harder later to make changes. If code was simply copied and pasted, then every instance would have to be modified to make even the smallest change.

Programming languages have ways to avoid code repition. for HTML, it is CSS: Cascading Style Sheets.
